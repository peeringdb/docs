# Do we still need public beta?
*July 21, 2026*

We will deploy updates to production, www.peeringdb.com, without a public beta period in August 2026.

Historically, we’ve had a three stage process for PeeringDB development:

1. New features and fixes are developed and tested internally
2. New features and fixes are publicly deployed to beta.peeringdb.com
3. New features go into production on www.peeringdb.com

That public beta testing period was important when [our API](https://docs.peeringdb.com/api_specs/) was less stable, or we made significant changes to the web UI. But things have changed.

We’ve noticed that very few people use [beta.peeeringdb.com](https://beta.peeringdb.com). And when we asked users from large and small networks, tool developers, and others, the overwhelming response was that people didn’t feel it was necessary.

So we’d like to change our process so we deploy improvements more quickly. And that’s why we’re running this test in August. We’ll use it to evaluate how we should refine our processes to improve our pace without irritating users.

You might notice a change to our announcements. As always, [release notes](https://docs.peeringdb.com/release_notes/) will be published.

If you love or hate this change, please let us know. We want your feedback.

If you have an idea to improve PeeringDB you can share it on our low traffic [mailing lists](https://docs.peeringdb.com/#mailing-lists) or create an issue directly on [GitHub](https://github.com/peeringdb/peeringdb/issues). If you find a data quality issue, please let us know at [support@peeringdb.com](mailto:support@peeringdb.com).

--- 

PeeringDB is a freely available, user-maintained, database of networks, and the go-to location for interconnection data. The database facilitates the global interconnection of networks at Internet Exchange Points (IXPs), data centers, and other interconnection facilities, and is the first stop in making interconnection decisions.
